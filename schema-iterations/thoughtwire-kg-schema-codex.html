Executive Verdict

Build the graph as a product-aware temporal causal knowledge graph, not as a pure tree. Use a tree-like spine for navigation and access control, but keep causal links as temporal graph edges with lag, evidence, confidence, and review state.

Graph Shape

Core Nodes

Business/Semantic

Tenant, IntelligenceProduct, Domain, Platform, MetricConcept, Metric, Formula, Dimension.

Metric stores product, domain, source_set, causal_role, card endpoint, series endpoint, threshold summary, policy summary.

Data/Experience

APISource, SourceProfile, Endpoint, Connector, DataAsset, Dashboard, DashboardView, UIComponent.

Chart registry belongs mostly on UIComponent and Formula, not inside Metric as a blob.

Governance/Social

Role, Person, Policy, Threshold, ApprovalRule, Action, Tool.

Role graph filters context before an agent sees any restricted metric or downstream effect.

Memory/Learning

Thoughtlet, DecisionCapsule, MonitoringContract, WakeCondition, LearningCandidate, PromotedMemory, GraphChangeProposal.

Capsules anchor to graph nodes but do not mutate causal truth directly.

Causal Evidence

CausalRelation, EvidenceEvent, GraphVersion.

Confidence is derived from an evidence ledger; deterministic formula edges can be pinned at confidence 1.0.

Product Seed

miq, ciq, piq, creative_iq, dc.

Tenant access is modeled with ENTITLED_TO and product-level RBAC.
