ThoughtWire · Neo4j · one DB per tenant · 2026-06-13

The business body an agent wakes into.

A causal knowledge graph — the stable map of what a business measures, what moves it, who may act, and what it has learned. This revision combines the full Claude blueprint with the lean Codex V1 node-details and applies five data-grounded merges. Part I ships first; Part II is the full vision.

Part I — V1what ships first

01 Verdict & V1 stance

The idea is right; three things must hold, and V1 is deliberately lean.

01

Not a tree, not one flat graph

A tree-like spine (Product → Domain → Metric) for orientation; a temporal DAG for causality; governance as overlay.

02

The hard part is edges

rare_seeds has 355 nodes, 4 edges (correlation, one spurious). Nodes are solved; trustworthy causal edges are the whole project.

03

Confidence is derived

Edge strength folds from an append-only evidence ledger (V2), never typed in; writes go through governed proposals.

Tenancy · no Tenant node

Each tenant gets its own Neo4j database. Tenant identity is database/runtime context — there is no Tenant node and no tenant_id field. This matches the per-tenant Snowflake USR_/RL_/WH_<SLUG> users/roles/warehouses and per-tenant databases (e.g. DB_RARE_SEEDS). If one-DB-multi-tenant is ever forced, reintroduce Tenant with composite constraints.

The five V1 merges — all validated against chart-registry.json, openapi.json, and the real rare_seeds shape:

−Tenant

Per-tenant DBs ⇒ tenant = runtime context, not a graph entity. Drops tenant_id everywhere.

→MetricConcept into Metric

No concept entity in the data; concept ≈ metric_base. Folded to concept_key/concept_name; cross-scope via ROLLS_UP_TO.

→Endpoint + Connector into Metric

rare_seeds already stores them flat: card_endpoint, series_endpoint, source_set, mart_source.

→Principal into Role

No principal in the data — only User {role}. App authenticates; graph needs only the Role (role_key claim).

±Type audit

Real values disagree with old types — type_confidence: high is categorical, booleans are yes/no, lists are |-delimited (§07).

Query patterns

// 1 · Role closure — assigned + inherited
MATCH (r:Role {role_key:$role_key, status:'active'})
OPTIONAL MATCH (r)-[:INHERITS_FROM*0..4]->(p:Role {status:'active'})
RETURN collect(DISTINCT r)+collect(DISTINCT p) AS role_scope;

// 2 · Allowed metrics — product/domain grants flow down; explicit deny wins in app logic
MATCH (r:Role {role_key:$role_key, status:'active'})
OPTIONAL MATCH (r)-[pm:CAN_VIEW|CAN_ACCESS_DOMAIN|CAN_ACCESS_PRODUCT]->(t) WHERE pm.effect='allow'
WITH collect({target:t, grant:pm}) AS grants
MATCH (m:Metric {status:'active'})
WHERE any(g IN grants WHERE
  (g.target:Metric AND id(g.target)=id(m)) OR
  (g.target:Domain AND (m)-[:BELONGS_TO]->(g.target)) OR
  (g.target:IntelligenceProduct AND (m)-[:PART_OF_PRODUCT]->(g.target)))
RETURN DISTINCT m.metric_uid, m.concept_name, m.product_id, m.data_classification;

// 3 · Agent answer context for one metric — only allowed facts before the LLM sees context
MATCH (r:Role {role_key:$role_key, status:'active'}),
      (m:Metric {metric_uid:$metric_uid, status:'active'})
WHERE EXISTS { MATCH (r)-[g:CAN_VIEW]->(m) WHERE g.effect='allow' }
OPTIONAL MATCH (m)-[:HAS_THRESHOLD]->(t:Threshold), (m)-[:GOVERNED_BY]->(p:Policy)
OPTIONAL MATCH (c:UIComponent)-[:VISUALIZES]->(m), (up:Metric)-[:CAUSES]->(m)
RETURN m, collect(DISTINCT t), collect(DISTINCT p),
       collect(DISTINCT c), collect(DISTINCT up.metric_uid) AS upstream;

// 4 · Safe edit check — require explicit edit permission before any mutation
MATCH (r:Role {role_key:$role_key})-[g:CAN_EDIT]->(t:Threshold {threshold_id:$tid})
WHERE g.effect='allow'
RETURN t.threshold_id, g.approval_required;

12 Confidence — derived, never typed in

Edge strength folds from an append-only ledger. This numeric confidence is distinct from the Metric's categorical causal_role_confidence.

# Jeffreys prior
α = 0.5,  β = 0.5
for supporting EvidenceEvent: α += weight
for refuting   EvidenceEvent: β += weight
confidence    = α / (α + β)
evidence_mass = α + β       # 0.8 from one guess ≠ 0.8 from forty outcomes
Evidence tierWeightNote
formulapinned 1.0deterministic edges sit outside the fold
prior / llm1imported / hypothesis — must be reviewed
observational2–5scaled by effect size, sample, FDR, lag
quasi_experimental5natural experiment / diff-in-diff
interventional8an approved action moved a lever, result followed
human · outcome10 · 8domain-expert confirmation · predicted-vs-actual reconciliation

The 4 rare_seeds correlation edges enter as CORRELATES_WITH + an observational event — never auto-promoted to CAUSES.
