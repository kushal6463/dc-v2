Compact causal KG with six operational nodes

The graph stores only the nodes needed for role-filtered agent context today: Metric, Dashboard, UIComponent, Policy, Threshold, and Role. Product, domain, metric concept, endpoint, connector, and user identity are fields or runtime context until they need graph-native traversal.

Tenant model Each tenant has a separate Neo4j database. Tenant identity is handled by database selection, app routing, and external audit logs, not by a Tenant node.

Role-Only RBAC

The app authenticates the user, resolves role_key, and only then reads graph context.

filter before context

View

CAN_VIEW_METRIC, CAN_VIEW_DASHBOARD, and CAN_VIEW_COMPONENT control read context.

Edit

CAN_EDIT_POLICY and CAN_EDIT_THRESHOLD gate mutation requests.

Approve

CAN_APPROVE_CHANGE handles high-impact changes and approval workflows.

Allowed Context Query

MATCH (r:Role {role_key: $role_key, status: 'active'})
MATCH (m:Metric {metric_id: $metric_id, status: 'active'})
WHERE EXISTS {
  MATCH (r)-[grant:CAN_VIEW_METRIC]->(m)
  WHERE grant.effect = 'allow'
}
OPTIONAL MATCH (c:UIComponent)-[:VISUALIZES]->(m)
OPTIONAL MATCH (d:Dashboard)-[:CONTAINS_COMPONENT]->(c)
OPTIONAL MATCH (m)-[:HAS_THRESHOLD]->(t:Threshold {status: 'active'})
OPTIONAL MATCH (m)-[:GOVERNED_BY]->(p:Policy)
RETURN m, collect(DISTINCT c), collect(DISTINCT d),
       collect(DISTINCT t), collect(DISTINCT p);

Edit Check

MATCH (r:Role {role_key: $role_key, status: 'active'})
MATCH (t:Threshold {threshold_id: $threshold_id, status: 'active'})
MATCH (r)-[grant:CAN_EDIT_THRESHOLD]->(t)
WHERE grant.effect = 'allow'
RETURN t.threshold_id, grant.approval_required;

V1 Edges

Keep edge names explicit. Most access logic lives on role permission edges.

relationship catalog
CONTAINS_COMPONENT

Dashboard -> UIComponent. Dashboard composition.

VISUALIZES

UIComponent -> Metric. Chart/card/table shows a metric.

HAS_THRESHOLD

Metric -> Threshold. Metric boundary.

GOVERNED_BY

Metric -> Policy. Policy scope.

EXPLAINS_THRESHOLD

Policy -> Threshold. Policy explains threshold intent.

INFLUENCES / CAUSES / CORRELATES_WITH

Metric -> Metric. Driver, causal, or association relationship.

INHERITS_FROM

Role -> Role. Permission inheritance.

CAN_VIEW_*

Role -> Metric/Dashboard/UIComponent. Visibility and traversal.

CAN_EDIT_*

Role -> Policy/Threshold. Mutation gating.

CAN_APPROVE_CHANGE

Role -> Policy/Threshold. Approval authority.

OWNS

Role -> Metric/Policy/Threshold/Dashboard. Accountability.
